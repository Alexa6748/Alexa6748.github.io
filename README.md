# КРИЗИС.EXE — сайт мероприятия

Статический сайт для GitHub Pages: лендинг с выбором «Спокойная жизнь / Движуха», распределение по мирам бизнес-симуляции, распределение по командам с точкой на карте и квест-трекер.

## Структура репозитория

```
/
├── .github/
│   └── workflows/
│       └── static.yml        ← workflow автодеплоя на GitHub Pages
├── assets/
│   └── map.jpg                ← сюда положите фото/схему площадки (пока отсутствует — есть заглушка)
├── css/
│   └── style.css              ← вся тема (цвета, шрифты, карточки, квесты)
├── js/
│   ├── app.js                  ← общие утилиты, localStorage-состояние участника
│   ├── worlds.js               ← 6 миров бизнес-симуляции (трек «Спокойная жизнь»)
│   └── teams.js                ← команды, точки на карте, квесты (трек «Движуха»)
├── index.html                  ← лендинг / выбор пути (easy / hard)
├── track-easy.html             ← трек «Спокойная жизнь»: распределение по мирам
├── track-hard.html             ← трек «Движуха»: команда, карта, квесты
└── README.md
```

## Как это работает (важно!)

Сайт **полностью статический** — без бэкенда и базы данных. Состояние каждого участника (выбранный путь, мир, команда, статус квестов) хранится **в localStorage его собственного браузера**, привязано к устройству, на котором он сканировал QR.

Это значит:
- ✅ Идеально подходит как MVP/демо и для распределения по мирам/командам на месте (детерминированный хэш от имени → команда/мир, одинаковый результат при повторном заходе).
- ⚠️ Не даёт организаторам единый экран «кто в какой команде» из административной панели — каждый видит только свой результат в своём телефоне.

### Если нужен общий учёт (кто куда попал, статистика по всем 500 участникам)
Нужен минимальный бэкенд. Самый быстрый вариант без написания сервера:
1. **Google Sheets + Apps Script** как приёмник форм — 30 минут настройки, кураторы видят таблицу в реальном времени.
2. **Firebase Realtime Database / Firestore** (бесплатный тариф) — если нужно живое обновление счётчика команд на самом сайте.
3. **Supabase** — то же самое, но с Postgres и SQL, если планируете развивать проект дальше.

Дайте знать, если нужно — подключу один из вариантов поверх текущей вёрстки.

## Настройка под своё мероприятие

- **Число команд**: в `js/teams.js` поменяйте `TEAM_SIZE` (сколько человек в команде) и `DEFAULT_REGISTERED` (сколько всего зарегистрировано на «Движуху») — число команд и точки на карте пересчитаются сами.
- **Карта площадки**: положите файл `assets/map.jpg` — точки команд лягут поверх неё автоматически. Без файла показывается текстовая заглушка, но точки всё равно считаются.
- **Тексты миров/квестов**: правьте `js/worlds.js` и `js/teams.js` — массивы `WORLDS` и `QUESTS`.
- **Цвета/шрифты**: всё в переменных `:root` в начале `css/style.css`.

## Деплой

1. Создайте репозиторий `<ваш-логин>.github.io` (или обычный репозиторий и включите Pages в Settings → Pages → Source: GitHub Actions).
2. Залейте содержимое этой папки в ветку `main`.
3. Workflow `.github/workflows/static.yml` задеплоит сайт автоматически при пуше.
4. Сайт появится по адресу `https://<ваш-логин>.github.io/`.

> В примере из вашего исходного workflow был шаг подмены `PLACEHOLDER_KEY` на секрет карты (2GIS). Текущая версия сайта карту через API не подключает (используется статичное фото + расчёт точек на клиенте), поэтому шаг убран. Если подключите интерактивную карту (2GIS/Google Maps API), верните шаг `sed` из своего примера и добавьте `TWO_GIS_KEY` в Settings → Secrets.
